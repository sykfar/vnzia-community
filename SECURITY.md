# Security Policy

Die Sicherheit und Privatsphäre der Nutzerinnen und Nutzer von Vnzia haben hohe Priorität.

## Sicherheitslücken nicht öffentlich melden

Bitte erstelle **kein öffentliches Issue**, wenn dein Hinweis eine mögliche Sicherheitslücke, den Schutz persönlicher Daten oder einen Missbrauchsweg betrifft.

Dazu gehören insbesondere:

- unbefugter Zugriff auf Konten oder Daten,
- Offenlegung privater Reise- oder Standortdaten,
- Umgehung von Berechtigungen,
- Authentifizierungs- oder Sitzungsprobleme,
- veröffentlichte Zugangsdaten, Tokens oder API-Schlüssel,
- Injection-, XSS-, CSRF- oder vergleichbare Angriffsmöglichkeiten,
- Schwachstellen in Uploads, Freigaben oder Importfunktionen,
- konkrete Schritte zur Ausnutzung einer Schwachstelle.

## Vertrauliche Meldung

Bitte nutze – sobald im Repository aktiviert – GitHubs Funktion **Privately report a security vulnerability** unter dem Reiter **Security**.

Falls diese Funktion noch nicht verfügbar ist, veröffentliche keine technischen Details. Erstelle stattdessen ein allgemeines Issue mit dem Titel:

`Security contact requested`

Der öffentliche Text sollte ausschließlich darauf hinweisen, dass du einen vertraulichen Meldeweg benötigst. Füge keine Beschreibung der Schwachstelle, Screenshots, URLs, Kontodaten oder Reproduktionsschritte bei.

## Inhalt einer vertraulichen Meldung

Eine hilfreiche Meldung enthält:

- eine kurze Zusammenfassung,
- betroffene Funktion oder Oberfläche,
- mögliche Auswirkungen,
- nachvollziehbare Reproduktionsschritte,
- gegebenenfalls eine vorgeschlagene Abhilfe,
- deine Einschätzung zur Dringlichkeit.

## Verantwortungsvolle Offenlegung

Bitte gib dem Projekt angemessene Gelegenheit, den Hinweis zu prüfen und gegebenenfalls zu beheben, bevor Details öffentlich gemacht werden.

Dieses öffentliche Community-Repository enthält keinen Sourcecode und keine interne technische Sicherheitsdokumentation.
